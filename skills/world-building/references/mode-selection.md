# Mode Selection

Choose the lightest mode that still creates the right decisions.

## `foundational`

Use when the user is early, the domain is unfamiliar, or the main risk is misunderstanding the problem space.

Good fits:
- vague ideas
- science, psychology, or behavior-heavy domains
- brand, narrative, education, or service concepts
- fictional settings where the world itself is the product

Typical scale:
- 6-10 lead agents
- 20-50 subagents
- 60-90 minutes

## `deep-research`

Default mode for most serious pre-build requests.

Good fits:
- software products and features
- category mapping before a build
- startup or strategy research
- evidence gathering before a PRD

Typical scale:
- 10-15 lead agents
- 40-80 subagents
- 75-120 minutes

## `world-building`

Use when the user explicitly wants a harness or says "before building".

Good fits:
- "deep research before building"
- "build the world first"
- reusable research operating systems
- launch-ready evidence-linked planning

Typical scale:
- 20-40 lead agents
- 80-200 subagents
- 2-3 hours

## Escalation Rule

Start in the chosen mode, then escalate only if the evidence shows the current mode is too shallow.
